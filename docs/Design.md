# Design Guide — AquaESG 절감 분석 대시보드

> 대상: **신고대상 건물주(이관리)** 가 1순위, 서울시 환경정책담당관(김주무관)이 2순위.
> 단일 Streamlit 페이지에서 "내 건물 절감액 / ROI / ESG 정량지표 / 인근 수요처"를 5분 안에 이해하게 만든다.
> 기준 문서: `docs/PRD.md` v0.3, `docs/기획서.pdf`, `docs/TSD.md` v0.2
> 작성일: 2026-05-18 | 디자인 방향: 공공 데이터 신뢰감 + 정책 보고서 수준의 정보 밀도 + 2026 한국 핀테크/공공 대시보드 정제감

---

## 1. 디자인 목표

대시보드는 보기 좋은 화면이 아니라, 다음 사용자가 의사결정 근거를 얻는 화면이다.

- **건물주 (이관리, 대형 아파트 관리소장)** — 1순위 페르소나:
  우리 단지가 감면 대상인지, 설비 투자하면 몇 년에 회수하는지, ESG 보고에
  쓸 정량 수치가 있는지를 _3분 안에_ 확인해야 한다.
- **서울시 환경정책담당관 (김주무관)** — 2순위 페르소나:
  분기 보고에 쓸 _근거 있는_ 매칭 가능량·절감액·활용률 시뮬레이션이 필요하다.

따라서 디자인의 1순위는 장식이 아니라 다음 네 가지다.

1. 사이드바에 건물명을 넣으면 5초 안에 "연 \_\_\_만원 절감 / \_\_\_년 회수 / \_\_\_tCO₂" 정량 카드가 차오른다.
2. 첫 화면에서 "서울에서 매일 \_\_\_ 톤이 버려진다" Epiphany를 5초 안에 이해시킨다.
3. 지도 + 사이드바 + KPI 카드의 정보 밀도를 _데이터 분석 도구_ 수준으로 유지한다.
4. 모든 수치에 출처와 산출 근거(공공데이터셋 ID + 산식)를 1급 UX로 노출해 그린워싱 가능성을 차단한다.

## 2. 핵심 디자인 원칙

### 원칙 1. 공공 데이터 도구처럼 보인다

화려한 SaaS 템플릿이 아니라, 서울시 정책 부서가 쓰는 분석 화면처럼 보여야 한다.

- 흰 배경 + 검정 텍스트 + 절제된 블루 포인트.
- 보라색 SaaS 그래디언트, AI 슬롭 일러스트, 큰 블러 배경은 사용하지 않는다.
- 모든 강조색은 의미(데이터값, 위험, 매칭)와 1:1 매핑된다.

### 원칙 2. 수치는 항상 단위와 출처를 동반한다

- 발생량은 항상 `톤/일` 또는 `톤/년`. 단위 없는 숫자는 카드에 두지 않는다.
- 화폐는 `원/년` 또는 `억원/년`. 천 단위 콤마를 항상 표시한다.
- 카드 하단에 한 줄 caption으로 산식과 근거 데이터셋을 노출한다 (예:
  `100% 재이용 시 추정치. 하수도요금 50% 감면 조례·전력 배출계수 0.4594 tCO₂eq/MWh 기준`).

### 원칙 3. 지도는 컨텐츠, 사이드바는 컨트롤

- 가운데 큰 지도는 데이터 자체. 마커 색은 의미(매칭됨/미매칭/수요처)에 고정.
- 사이드바는 검색·필터·세부카드. 인터랙션이 일어나는 모든 컨트롤은 사이드바.
- 사이드바와 메인 영역을 카드 안에 다시 카드로 감싸지 않는다.

### 원칙 4. 페르소나별 정보 우선순위가 다르다 — 건물주 우선

| 페르소나     | 첫 시선                          | 두 번째 시선             | 세 번째 시선                   |
| ------------ | -------------------------------- | ------------------------ | ------------------------------ |
| **건물주 (1순위)** | 사이드바 건물 검색 + 일 발생량 | 사이드바 ROI/PBP/CO₂ 카드 | ESG 리포트 다운로드 + 지도(수요처) |
| 김주무관(시) | KPI 카드 4종 (Epiphany)          | 지도 매칭 흐름           | 매칭 솔루션 요약               |

→ 사이드바 건물 카드는 페이지 좌측 상단에 항상 노출, 검색 후 즉시 ROI 블록이 차오른다.
→ KPI 카드는 메인 영역 상단, 지도는 화면 중앙.

### 원칙 5. 모든 수치는 소스를 클릭으로 펼친다 (2026 커뮤니티 트렌드)

수치 옆 `ⓘ` 아이콘 또는 caption 행을 누르면 산식과 데이터셋 ID(OA-15607 등)를 인라인 팝오버로 표시한다. ESG 시대의 그린워싱 방지는 _소스 시테이션을 1급 UX 요소_ 로 다루는 것에서 시작한다.

## 3. Visual System

### 3-1. 색상

전체 인상은 흰 표면 + 검정 텍스트 + 1개의 절제된 강조색.

| 용도              | 토큰                | 값        | 사용처                          |
| ----------------- | ------------------- | --------- | ------------------------------- |
| 배경              | `--color-bg`        | `#F7F7F5` | 페이지/사이드바 외부            |
| 표면              | `--color-surface`   | `#FFFFFF` | 카드 배경, 마커 외곽선          |
| 기본 텍스트       | `--color-text`      | `#111111` | 카드 값, 헤딩                   |
| 보조 텍스트       | `--color-muted`     | `#666A70` | 라벨, caption, 단위             |
| 경계선            | `--color-border`    | `#E4E4E0` | 카드 분리선                     |
| Primary           | `--color-primary`   | `#0071E3` | 강조 링크, 매칭된 공급처 마커   |
| 데이터 강조(긍정) | `--color-data`      | `#1D7F5F` | 절감액, 탄소절감, 활용률 값     |
| 위험/미매칭       | `--color-risk`      | `#B54708` | (예약: 부정 KPI, 미매칭 마커)   |

지도 마커 색은 별도로 고정:

| 의미                       | 색                                 |
| -------------------------- | ---------------------------------- |
| 매칭된 공급처 (신고대상)   | `#3b82f6` (Primary 계열)           |
| 미매칭 공급처              | `#ef4444` (Risk 계열)              |
| 수요처 - 공원              | `#f97316`                          |
| 수요처 - 도로              | `#fb923c`                          |
| 매칭 흐름선                | `#22c55e` (Data 계열)              |

사용 규칙:
- 한 화면에서 데이터 강조색(`--color-data`) 사용은 _수치 의미_ 가 있을 때만.
- 배경 그래디언트는 사용하지 않는다. 섹션 구분이 필요하면 1px 경계선.
- 색상만으로 상태를 구분하지 않는다. 항상 라벨/배지를 동반.

### 3-2. 타이포그래피

본문은 **Pretendard Variable** (2026 한국 핀테크/공공 대시보드 사실상 표준).
숫자는 SF Pro 또는 Inter의 _tabular-nums_ 변형으로 정렬한다.

```css
font-family:
  "Pretendard Variable", Pretendard,
  -apple-system, BlinkMacSystemFont, "Segoe UI",
  "Apple SD Gothic Neo", "Noto Sans KR", sans-serif;
font-feature-settings: "tnum" 1, "ss01" 1;  /* tabular nums */
```

Streamlit에서는 `app/components/fonts.py` 에서 `<link>` 1줄로 CDN 주입.
Pretendard가 로드되지 않은 경우 시스템 폰트로 자연스럽게 폴백하므로 FOUT 부담은 낮다.

| 역할             | 크기 | 굵기 | 줄간격 | 비고                          |
| ---------------- | ---: | ---: | -----: | ----------------------------- |
| 페이지 H1        | 24px |  700 |   1.25 | 좌측 상단 1줄                 |
| 서브 카피        | 14px |  400 |   1.45 | H1 바로 아래 1줄, muted       |
| KPI 라벨         | 11px |  600 |   1.20 | uppercase, letter-spacing .05 |
| KPI 값           | 22px |  700 |   1.15 | 천 단위 콤마                  |
| 건물 카드 이름   | 15px |  700 |   1.35 |                               |
| 건물 카드 주소   | 12px |  400 |   1.35 | muted                         |
| 건물 카드 발생량 | 28px |  700 |   1.15 | data 색                       |
| ROI 값           | 16px |  700 |   1.20 | data 색                       |
| Caption          | 10px |  400 |   1.40 | 출처/산식, muted              |

규칙:
- 글자 자간(letter-spacing)은 라벨류에만 `.04~.05em` 적용. 본문은 `0`.
- 본문은 명사 나열보다 짧은 동사형 문장 (예: `발생량을 보여줍니다`).
- 숫자는 항상 한 줄 안에 단위까지 들어오게 폭을 조정. 줄바꿈 금지.

### 3-3. 레이아웃 — 정제된 bento 감각

2026 커뮤니티의 _bento grid_ 트렌드는 가져오되, SaaS 마케팅 화면의 거대한 셀과 그래디언트는 배제한다. AquaESG의 bento는 "정책 보고서가 격자 위에 인쇄된 느낌"에 가깝다.

| 항목             | 기준                                       |
| ---------------- | ------------------------------------------ |
| Streamlit layout | `wide`                                     |
| 사이드바 폭      | Streamlit 기본 (≈ 21rem)                   |
| 지도 높이        | 620px (fixed)                              |
| KPI 그리드       | desktop 4열 / ≤960px 2열 / ≤640px 1열      |
| ROI 카드 그리드  | 2x2 (절감액·PBP·CO₂·재이용량)              |
| 매칭 요약 그리드 | 3열 고정                                   |
| 카드 반경        | 10px (이전 8px → bento 정렬감)             |
| 카드 패딩        | 16~20px                                    |
| 카드 간 gap      | 12px (이전 1px border → 호흡 확보)         |
| 경계선 굵기      | 1px, hairline 색 `#E8E8E4`                 |
| 그림자           | 사용 안 함 — 경계선만으로 분리             |

레이아웃 규칙:
- 메인 영역: H1 → KPI 4종 → ROI/매칭 요약 → 지도. 위에서 아래로 정보 우선순위.
- 사이드바: 건물 검색 → 건물 카드 → ROI 블록 → 매칭 반경. _건물주가 첫 화면 좌측만 봐도 의사결정이 끝나야 한다._
- 카드 안에 카드를 겹치지 않는다. 그리드 셀이 카드 역할.
- 모바일은 1차 지원 대상이 아니다. 단, KPI 그리드는 `≤640px` 에서 1열로 폴드되어 깨지지 않게 한다.

### 3-4. 모션

분석 도구는 모션을 거의 쓰지 않는다.

- Streamlit 기본 컴포넌트 트랜지션 외 추가 모션 없음.
- 지도 마커 진입 애니메이션 없음.
- 카드 hover lift/그림자 변화 없음 (정보 정적 표시가 신뢰감).
- `prefers-reduced-motion` 사용자에게는 동일 (기본부터 모션 최소).

## 4. 페이지 구조

### 4-1. 헤더

- H1: `AquaESG — 서울 유출지하수 절감 분석`
- 서브 카피 1줄: `서울시 공공데이터 기반 건물별 절감액 · ROI · ESG 정량 리포트`
- 우측 보조 영역: 데이터 출처 배지 4종 (환경 · 기상 · 에너지 · 인구) — PRD §8 가점 시각화

금지:
- 추상적인 환경 사진 배경 위에 텍스트 올리는 방식
- "AI로 모든 물 문제 해결"같은 과장 카피
- 헤더 안에 3개 이상의 링크/CTA

### 4-2. Epiphany KPI 카드 4종

PRD §9 FR-04를 시각화한 4개 카드. 가로로 1열 배치 (`≤960px` 에서 2x2 폴드).

| 카드        | 라벨                | 값 형식              | 강조         | 캡션 (출처)                    |
| ----------- | ------------------- | -------------------- | ------------ | ------------------------------ |
| 1           | 오늘 하수도행       | `153,000 톤`         | 기본         | OA-15607 + 일평균 환산         |
| 2           | 연 절감 가능액      | `223 억원`           | data 색      | 하수도 50% 감면 조례 기준       |
| 3           | 탄소 절감           | `10,255 t-CO₂`       | 기본         | 0.4 kWh/톤 × 0.4594 tCO₂eq/MWh |
| 4           | 활용률              | `7.9% → 100%`        | data 색      | 미활용 75.6% 환산              |

규칙:
- 라벨은 11px UPPERCASE muted, 값은 22px 검정/data, 캡션은 10px muted.
- 각 카드 우측 상단에 `ⓘ` (12px muted) — 클릭 시 산식·데이터셋 ID 팝오버.
- 활용률 카드는 항상 `현재% →` 접두로 Epiphany 의미를 강화.

### 4-3. 건물 ROI 블록 (검색 후 노출)

PRD §9 FR-03 핵심. 사이드바에서 건물이 선택된 직후 메인 상단의 KPI 카드 아래 4셀 그리드로 차오른다.

| 셀 | 라벨 | 값 | 출처 |
| --- | --- | --- | --- |
| 1 | 연 재이용 가능량 | `387,000 톤/년` | FR-02 LightGBM 예측 합산 |
| 2 | 연 하수도 절감 | `7,740 만원/년` | 재이용량 × 400원 × 50% |
| 3 | 연 탄소 절감 | `71.1 tCO₂eq/년` | 0.4 kWh/톤 × 0.4594 |
| 4 | 투자 회수 기간 | `4.2 년` (PBP) | capex ÷ 연 절감액 |

각 셀 하단 12px 행에 `자세히 ▾` — 펼치면 사용된 단가·계수가 표로 노출되어 _그린워싱 검증_ 이 가능하다.

### 4-4. 매칭 솔루션 요약 (3셀)

- 일 매칭량 / 목적함수 (백만원/일) / 수요 충족률.
- 사용자가 사이드바에서 반경을 바꾸면 즉시 재계산 표출.

### 4-5. 지도 (Kakao Maps)

높이 620px 고정. 4개 레이어:

1. **공급처 마커** — 신고대상 건물(21층 ∨ 10만㎡). 색은 매칭/미매칭 분기.
   크기는 `daily_avg_supply_ton / 90`을 10~24px로 클램프 (큰 발생량일수록 큼).
2. **수요처 - 공원** — orange `#f97316`. 크기는 `area_m2 / 180000` 6~16px.
3. **수요처 - 도로** — orange `#fb923c`, 작은 6px 고정.
4. **매칭 흐름선** — green `#22c55e`, 굵기 `ton_per_day / 30` 2~7px.

마커 클릭 시 InfoWindow:
- 공급처: 건물명, 발생량(톤/일), 수질등급, 신고상태.
- 수요처: 이름 + 면적 또는 연장.

검색 시:
- 건물명 입력 → 첫 매칭 마커로 자동 센터링 + InfoWindow 자동 오픈.

### 4-6. 사이드바 (건물주 1순위 동선)

순서: 검색 → (검색 결과 카드 + ROI 블록) → ESG 리포트 다운로드(Phase 2) → 매칭 반경.

**건물 검색 입력**
- 라벨: `건물 검색` (UPPERCASE muted, 11px)
- placeholder: `예: 서울 광역자원순환센터`
- Streamlit `text_input(label_visibility="collapsed")`로 시각적 라벨은 위에 직접 둔다.

**건물 카드 (검색 결과)**
- 흰 배경 카드 안에 다음 순서:
  1. 건물명 (15px bold)
  2. 주소 (12px muted)
  3. 일 발생량 (28px data 색, 단위 `톤/일`)
  4. 메타 row: 수질등급 / 신고상태 / 지하층 수 (각 11px 라벨 + 13px 값)
  5. 신고대상 배지 (해당 시): 작은 녹색 배지 `신고대상 ✓`
  6. **재이용 목적 셀렉트** (PRD §6.1 Step 2): `냉방 / 청소 / 조경`
  7. **ROI 블록**: 1px 상단 경계선 → 2열 그리드 (메인 영역 §4-3과 동일 산식, 압축 표시)
     - 연 하수도 절감 (만원/년, data 색)
     - 연 탄소 절감 (tCO₂/년, data 색)
     - PBP (년, data 색)
     - 연 재이용량 (톤/년)
  8. Caption (10px muted): `산식·출처 보기 ▾` — 펼치면 데이터셋 ID + 단가·계수

**ESG 리포트 (Phase 2)**
- `리포트 PDF 다운로드` 버튼 (FR-06). MVP에서는 비활성화 + 툴팁 `Phase 2 — 시상식 후 공개`.
- 활성화 시 GRI 303·305 양식이 매핑된 1-페이지 요약 + 산식 부록 PDF.

**매칭 반경 라디오**
- 옵션: 500m / 1km / 2km. 기본 1km.
- horizontal radio. 변경 시 즉시 KPI/지도 갱신.

### 4-7. 푸터

- 1줄 caption: `데이터 출처: 서울 열린데이터광장 (OA-15607 외 6종) · 기상청 ASOS Open API · 기상청·농진청 SPEI`
- 라이선스 표시 의무(공공데이터 이용약관) 충족.
- 모델 버전 표기: `AquaESG v0.x · LightGBM residual v1 · PuLP CBC`

## 5. 컴포넌트 규칙

### Buttons / Controls

- Streamlit 네이티브 컨트롤만 사용. 커스텀 버튼 컴포넌트 도입 금지 (MVP 범위 외).
- 사이드바 컨트롤은 검색 1개 + 반경 라디오 1개로 끝낸다. 추가하지 않는다.

### Badges

- `신고대상 ✓` 배지: 녹색 약한 톤 `#E6F4EE` 배경, `#1D7F5F` 텍스트.
- 헤더 우측 데이터 분야 배지 4종 (환경/기상/에너지/인구): 모두 같은 muted 톤 (`#F0F0EC` 배경, `#444` 텍스트). 색으로 강조하지 않는다.
- 다른 상태 배지를 새로 만들지 않는다 (정보 과부하 방지).

### Source citation 컴포넌트 (2026 트렌드)

- 수치 옆 `ⓘ` 아이콘 (12px muted) — 클릭 시 인라인 팝오버.
- 팝오버 내용: `산식 1줄` + `출처: 데이터셋명 (ID)` + `갱신 주기`.
- 절대 모달로 띄우지 않는다 — 컨텍스트 단절을 만들지 않는다.

### KPI / Match 카드

- 카드의 시각적 단위는 _그리드 셀_. 셀끼리는 1px border만.
- 셀 내부 padding 14~18px, 라벨/값 사이 4px.

### 지도 InfoWindow

- 흰 배경, 패딩 6px 10px, 폰트 12px line-height 1.7.
- 첫 줄: 건물명/수요처명 (bold).
- 둘째~넷째 줄: 핵심 수치 + 단위.

## 6. 카피 톤

차분하고 구체적. 정책 문서와 비슷한 톤.

좋은 문장:
- `서울시 공공데이터 기반 유출지하수 공급처 · 수요처 매칭 분석`
- `100% 재이용 시 추정치. 하수도요금 50% 감면 조례 기준`
- `검색 결과가 없습니다.`

피할 문장:
- `AI가 자동으로 모든 절감액을 계산합니다!`
- `완전 자동화된 ESG 솔루션`
- `클릭 한 번으로 모든 분석이 끝납니다.`

KPI 라벨은 명사형 짧게:
- `오늘 하수도행` (X `오늘 하수도로 흘려보낸 양`)
- `연 절감 가능액` (X `연간 절감 가능한 금액`)

## 7. 접근성 및 성능 기준

- 텍스트 대비는 WCAG AA 이상.
- 사이드바 입력은 항상 라벨이 위에 보이게 한다. placeholder만으로 라벨 대체 금지.
- 색상만으로 상태 구분 금지. 마커 색에는 라벨 또는 InfoWindow 텍스트 동반.
- 지도 초기 로딩 ≤ 5초 (TSD §8).
- 반경 변경 후 KPI/매칭 갱신 ≤ 2초 (캐시된 사전계산 솔루션 3종 사용).
- 무료 티어 Streamlit Cloud sleep 깨움은 시상식 전날 1회 트리거.

## 8. 디자인 QA 체크리스트

구현 완료 전 다음을 확인한다.

- [ ] 첫 화면에서 4개 KPI 카드가 한 줄(또는 2x2)로 보인다.
- [ ] 헤더 H1 1줄 + 서브 카피 1줄 + 데이터 분야 배지 4종이 한 행에 들어온다.
- [ ] Pretendard 가 로드되며, 미로드 시 시스템 폰트로 폴백된다.
- [ ] 모든 수치 카드에 단위가 동반된다 (톤/일, 억원/년, t-CO₂/년, 년 등).
- [ ] 모든 추정 카드에 `ⓘ` 아이콘 또는 산식·근거 caption이 1줄로 보인다.
- [ ] 건물 검색 시 메인 영역에 ROI 4셀(연 재이용량/하수도 절감/탄소 절감/PBP)이 차오른다.
- [ ] 사이드바 건물 카드에 재이용 목적 셀렉트(냉방/청소/조경)가 노출된다.
- [ ] ESG 리포트 버튼은 MVP에서 비활성화 상태 + `Phase 2` 툴팁이 보인다.
- [ ] 지도 마커가 PRD FR-01의 4계층(공급/공원/도로/흐름)을 모두 표현한다.
- [ ] 매칭 vs 미매칭 공급처 색이 분명히 구분된다.
- [ ] 건물 검색 시 사이드바 카드 + 지도 자동 센터링 + 메인 ROI 블록이 동시에 갱신된다.
- [ ] 사이드바 컨트롤이 검색 1 + 재이용 목적 1 + 반경 라디오 1 + (Phase 2 ESG 버튼 1) 이상으로 늘어나지 않는다.
- [ ] 푸터에 데이터 출처 7종 + 모델 버전이 표시된다.
- [ ] 색상만으로 상태(매칭/미매칭, 위험)를 구분하지 않는다.
- [ ] 페이지 어디에도 보라색 SaaS 그래디언트나 AI 슬롭 일러스트가 없다.
- [ ] bento 그리드의 셀 간 gap(12px)이 적용되되, 셀 내부에 그림자/그래디언트가 없다.

## 9. 구현 중 의사결정 기준

디자인 선택이 애매하면 다음 순서로 판단한다.

1. 서울시 정책 담당자가 분기 보고에 _그대로 캡처해 붙일 수 있는가_?
2. 모든 수치에 출처·산식이 보이는가 (그린워싱 방지)?
3. 정보 밀도가 _분석 도구_ 수준인가, 아니면 마케팅 페이지처럼 비어 보이는가?
4. 카드/색/배지를 더하는 대신 _빼서_ 같은 정보를 전달할 수 있는가?
5. PRD/기획서/TSD 스코프 밖 기능을 새로 끌어들이지 않는가?

위 질문 중 3개 이상에 답하지 못하는 요소는 제거한다.
