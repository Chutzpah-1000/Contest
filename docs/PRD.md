## 1. 제품 개요

### Epiphany — 제품의 출발점

> 서울 광역자원순환센터 한 곳에서 하루 **1,326톤**, 창동 문화산업단지에서 하루 **1,136톤**의 깨끗한 지하수가 매일 하수도로 그냥 흘러간다.
> 같은 도시에서 공원관리사업소는 매년 수십억 원의 상수도 요금을 내며 **똑같은 비음용 용도**(조경·살수)에 정수된 수돗물을 쓴다.
>
> 이건 _물이 부족하지 않아서_ 안 풀리는 문제가 아니라, **공급자와 수요자가 서로의 존재를 모르기 때문에** 안 풀리는 문제다.

이 깨달음을 정량으로 시각화해 의사결정자(서울시)에게 전달하는 도구가 본 제품이다.

### 솔루션 한 줄

서울시 공공데이터 기반으로, 유출지하수 **공급처(신고대상 건물)** 와 비음용수 **수요처(공원·도로·시설)** 를 지도 위에서 매칭하고, AI 수요예측·최적화로 **재이용 가능량과 절감 가치를 정량화**하는 B2G 분석 SaaS.

### PMF 아키타입 진단: Hard Fact

| 차원               | 진단                                                                         |
| ------------------ | ---------------------------------------------------------------------------- |
| Customer mindset   | "유출지하수는 그냥 흘려보내는 것" — 서울시 CSV 활용 칸 대부분 공란           |
| Category           | 비존재. 매칭 플랫폼 자체가 시장에 없음                                       |

## 2. 왜 지금까지 안 풀렸나 (Hard Fact 분석)

### 2.1 4가지 관성

1. **부서별 KPI 단절** — 신고는 환경부, 활용은 서울시 공원녹지과·도로과, 통계는 상수도사업본부. 한 명이 의사결정할 단위가 없음.
2. **책임소재 회피** — 비음용수라도 사고 발생 시 책임 주체 불명확. 신고만 하고 하수로 보내는 게 가장 안전한 선택지.
3. **운반/배관 부재** — 발생지·사용지가 떨어져 있으면 탱크로리·임시배관 비용이 절감액을 초과 (→ 본 MVP는 _반경 N km_ 내 케이스만 매칭).
4. **데이터 비대칭** — 시행사는 옆 공원이 얼마 쓰는지 모르고, 공원은 옆 빌딩에서 얼마 나오는지 모름.

### 2.2 카운터-아규먼트 정면 응답

> **"한국은 물이 풍부해서 유출지하수를 굳이 쓸 필요가 없다."**
> — 한국지질연구원 박사 (2026 인터뷰)

거시(전국 수자원 총량) 관점에서는 맞다. 하지만 우리 제품의 가치 근거는 다른 5축에 있다:

| 축              | 근거                                                                         |
| --------------- | ---------------------------------------------------------------------------- |
| **비용**        | 광역상수도 요금은 매년 인상. 비음용 용도에 정수 비용을 지불할 필연성 없음    |
| **외부효과**    | 폐수처리비는 발생자(건물주)가 부담. 흘려보내는 것도 *비용*이지 *공짜*가 아님 |
| **도시 물순환** | 하수도로 빠지는 지하수는 도시 미세물순환을 끊어 열섬·지반침하 유발           |
| **ESG/탄소**    | 정수→공급→재정수 펌핑 에너지 절감. 서울시 2050 탄소중립 KPI 정량화 가능      |
| **비상수원**    | 2022 남부 가뭄, 광역상수도 단일실패점 리스크. 분산형 수원의 보험 가치        |

→ 우리의 응답: **"물이 풍부해도 _정수된_ 물은 풍부하지 않다."** 그리고 본 제품은 _전국_ 물 부족이 아니라 *도시 부서별 비용·KPI*를 푼다.

> **"유출지하수 활용은 어렵다."**
> — 기후환경에너지부 (2026 인터뷰)

이 답변 자체가 문제 정의의 정확성을 입증함. _어렵다 = 해결 가치 있다._ §6 MVP는 이 어려움 중 **정보 비대칭** 한 축만 해소한다 (Hard Fact path는 처음부터 모두 풀려고 하지 않음).

## 4. Wedge 고객

### 4.1 1순위: 서울시 환경정책담당관 / 상수도사업본부 정책기획팀 (B2G)

**선택 근거**:

- 공급(신고대상 21층·10만㎡ 건물)과 수요(공원·도로·시설)가 모두 **서울시 행정 관할** → 한 의사결정자가 양쪽을 움직일 수 있는 유일한 위치.
- 재이용률 KPI를 가지지만 측정·증명 도구가 부재 (= 본 PRD가 그 도구).
- B2G 예산 항목: _데이터 분석 용역_, *정책 컨설팅*으로 연 단위 발주 가능 → B2B SaaS 결제 절차를 우회.
- 사용자(=결제자)가 1명이라 1인 영업으로도 PoC 가능.

**1차 컨택 가설**: 빅데이터 경진대회 창업 부문 시상식이 서울시청 8층에서 개최(2026-07-06) → 그 자리가 자연 컨택 포인트.

- 신축 신고대상 건축물 시행사 (이용계획서 의무 시점에 자동 진입)

## 7. AI/ML 명세 (창업 부문 핵심 어필)

> **솔직한 자기제한**: "전세계 AI 전문가급" 단일 모델은 1인 D-3에 불가능.
> 대신 채택한 패턴은 **도메인 지식 베이스라인 + ML 잔차 보정**으로, NeurIPS _Climate Change AI_ 워크숍 등에서 정식 인정되는 접근. 도메인 신뢰도와 정확도를 동시에 잡는 구조 → 발표에서 강조 포인트.

### 7.1 수요예측 모델

**과업**: 공원·도로 단위, **월별 비음용수 수요량(톤/월) 예측**

**입력 변수**

- _정적_: 면적, 식재 유형(잔디/관목/교목), 노면 길이, 좌표
- _동적_: 월, 강수량, 평균기온, 토양수분(Soil Moisture), 가뭄지수(SPEI), 미세먼지(살수 트리거)
- _외생_: 행사 일정, 휴일

**모델 트랙 (앙상블)**

1. **수문학 베이스라인** — 기준증발산량 ET₀ (Penman-Monteith) × 작물계수 × 면적
2. **LightGBM Residual** — 베이스라인 잔차를 ML로 보정 (도메인 + ML 결합)
3. _(Stretch)_ Temporal Fusion Transformer — 시간 여유 시

**검증**

- Walk-forward CV (월 단위 시차 검증)
- 메트릭: MAPE, RMSE, sMAPE
- 설명력: SHAP 값으로 변수 중요도 시각화 (심사 어필)
- 베이스라인 비교: 단순 월별 평균 대비 MAPE 개선폭

**현실 목표**: MAPE 20~30% (Demo 단계)

### 7.2 매칭 최적화

**문제 정의**: 가중 이분그래프 매칭 with capacity (Capacitated Bipartite Assignment)

**결정 변수**: x_ij = 공급처 i → 수요처 j 일배분량(톤/일)

**제약**

- Σ_j x_ij ≤ S_i (공급량 한도)
- Σ_i x_ij ≤ D_j (수요량 한도, §7.1 모델 출력)
- x_ij = 0 if dist(i,j) > R (반경 제약, 사용자 슬라이더)
- x_ij = 0 if 수질등급(i) ∉ 허용범위(j)

**목적함수**
maximize Σ x_ij × (상수도단가 − 추정운반비\_ij) − λ × 미충족수요

**알고리즘**

- N < 1000: PuLP + CBC (오픈소스 ILP)
- 중규모: Google OR-Tools Min Cost Flow
- 베이스라인: 그리디 최근접 매칭 → ILP 대비 절감액 차이로 최적화 가치 증명

**검증**

- 그리디 vs ILP 절감액 비교
- λ, R 민감도 분석
- 시각화: 현재 활용률 0% → 시스템 적용 시 \_\_\_% 시뮬레이션
